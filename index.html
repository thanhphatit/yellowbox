Greybox

Unified Hub for DevOps Scripts, Tools, and Helm Charts

About

Greybox is my unified repository — a single, structured hub that hosts every tool, script, and Helm chart I’ve built or curated. Instead of scattering small projects across multiple repos, everything here lives together, neatly categorized for clarity and scalability.

Each directory represents a functional layer of my personal DevOps ecosystem — from Kubernetes deployments to automation utilities and experiment tools.

Current Repository Structure

.greybox/ ├── CNAME ├── index.html ├── README.md ├── helm/ │ └── charts/ ├── tools/ │ ├── bash/ │ └── python/

This structure will continue expanding as more toolsets and modules are added.

Folders Overview

  • helm/ – Contains Helm-related assets and Kubernetes charts.
  • charts/ – Subdirectory under Helm for organized chart sets.
  • tools/bash/ – Shell scripts, cleanup utilities, and automation routines.
  • tools/python/ – Python-based utilities or APIs for system tasks or integrations.
  • index.html – This very overview page, used for GitHub Pages or documentation landing.
  • CNAME – Custom domain binding for the GitHub Pages site.

Philosophy

“Build once, reuse everywhere.” Every script and chart here is designed with reusability, observability, and minimal configuration in mind. The repo is built to grow — whether for internal automation, Helm packaging, or simple CLI utilities.

Future Expansion

The structure is modular — easily extensible to include:

  • /apps/ – Application prototypes or microservices.
  • /infra/ – Infrastructure definitions (Terraform, Ansible, etc.).
  • /scripts/ – Cross-language or mixed orchestration scripts.
  • /docs/ – Reference notes, design documents, or usage manuals.
